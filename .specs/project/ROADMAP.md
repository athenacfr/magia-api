# Roadmap

## Phase 1: Foundation — Config & Core

**Goal**: Define the config shape, schema resolution, core orchestration, runtime client, and plugin system.

### F-001: Unified Config (`defineConfig`)
A single `magia-api.config.ts` that declares all API connections.

```typescript
// magia-api.config.ts
import { defineConfig, tanstackQuery } from 'magia-api'

export default defineConfig({
  apis: {
    petstore: {
      type: 'rest',
      schema: 'https://petstore3.swagger.io/api/v3/openapi.json',
      plugins: [tanstackQuery()],
    },
    payments: {
      type: 'rest',
      schema: './schemas/payments.openapi.yaml',
    },
    internal: {
      type: 'rest',
      schema: async () => {
        const res = await fetch('https://internal.api/spec', { headers: { auth: '...' } })
        return res.text()
      },
      plugins: [tanstackQuery()],
    },
    legacy: {
      type: 'rest',
      schema: {
        command: './scripts/fetch-legacy-spec.sh',
        output: './schemas/legacy.json',
      },
    },
    github: {
      type: 'graphql',
      schema: 'https://api.github.com/graphql',
      documents: './src/api/graphql/**/*.graphql',
      plugins: [tanstackQuery()],
    },
    cms: {
      type: 'graphql',
      schema: async () => {
        const { Storage } = await import('@google-cloud/storage')
        const [content] = await new Storage().bucket('bucket').file('schema.graphql').download()
        return content.toString()
      },
      documents: './src/api/graphql/**/*.graphql',
    },
  },
})
```

**Per-API config shape:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `type` | `'rest' \| 'graphql'` | yes | API type |
| `schema` | `string \| () => Promise<string> \| { command, output }` | yes | Schema source |
| `documents` | `string \| string[]` | graphql only | Glob for `.graphql` operation files |
| `plugins` | `MagiaPlugin[]` | no | Magia plugins (tanstackQuery, etc.) |
| `schemaWatch` | `boolean` | no | Override schema watching (smart defaults) |
| `schemaCache` | `'disabled' \| { ttl: string }` | no | Override schema caching (smart defaults) |
| `operationName` | `(method: string, path: string, operationId?: string) => string` | no | Custom operation naming (default: use `operationId`, fallback: Hey API auto-generated) |

**Global config:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `dtsPath` | `string` | no | Path for generated `.d.ts`. Default: `src/magia-api.d.ts` if `src/` exists, else `./magia-api.d.ts` |

No output path config — everything goes to `node_modules/.magia/` (hidden) and `src/magia-api.d.ts` (gitignored).

### F-002: Schema Resolution
Flexible schema input supporting multiple source types:
- **URL** (`string`): Fetch from HTTP/HTTPS endpoint
- **Local file** (`string`): Read from filesystem (JSON, YAML, .graphql)
- **Async function** (`() => Promise<string>`): User-provided function that returns schema text
- **Script** (`{ command, output }`): Run a shell command, then read the output file
- Cloud storage (GCS, S3, etc.) handled via async functions or scripts

**Schema watch & cache** (smart defaults based on source):

| Source | `schemaWatch` default | `schemaCache` default |
|--------|----------------------|----------------------|
| Local file | `true` (file watcher) | `disabled` |
| `localhost` / `127.*` / `10.*` / `192.168.*` | `true` (poll + hash compare) | `disabled` |
| Remote URL | `false` | `{ ttl: '1h' }` |
| Async function | `false` | `{ ttl: '1h' }` |
| Script | `false` | `{ ttl: '1h' }` |

**Env var override for schemas** (`MAGIA_<API>_SCHEMA`):
```bash
MAGIA_PETSTORE_SCHEMA=./schemas/petstore-frozen.json
```

**Error recovery (dev only):**
On codegen error, auto-refetch schema (bust cache) and retry once. In CI/build: fail immediately.

### F-003: Codegen Engine
Opinionated — no user configuration of underlying codegen tools.

**REST APIs** — uses Hey API SDK internally:
- `@hey-api/typescript` → TypeScript types from OpenAPI schemas
- Manifest entries generated by magia from parsed spec (method, path, params, content-type)
- Data transformers built from OpenAPI `format` fields (date-time → Date, etc.)
- File upload operations auto-detected from `multipart/form-data` content-type

**GraphQL APIs** — uses graphql-codegen SDK internally:
- `@graphql-codegen/client-preset` → TypeScript types + compiled document strings
- Manifest entries generated by magia (query/mutation/subscription documents, variables)

**Runtime clients** (internal dependencies):
- `@hey-api/client-fetch` → HTTP client for REST operations
- `graphql-request` → HTTP client for GraphQL queries/mutations + SSE subscriptions

**Output** — all hidden:
```
node_modules/.magia/
├── manifest.ts                ← operation registry, bundled via Vite virtual module
├── internals/
│   ├── petstore/types.d.ts    ← Hey API type output
│   ├── github/types.d.ts      ← graphql-codegen type output
│   └── github/documents.ts    ← compiled GraphQL document strings
├── transformers.ts            ← data transformers from spec format fields
├── schemas/                   ← cached remote schemas
└── checksums.json             ← change detection

src/magia-api.d.ts             ← type augmentation (gitignored)
```

**Manifest bundling**: The manifest and GraphQL documents can't be read from `node_modules` at runtime in the browser. The Vite plugin resolves a virtual module (`virtual:magia-manifest`) that inlines the manifest into the bundle. `createMagia()` imports from this virtual module internally.

Users never see or configure internals. Zero visible generated files.

### F-004: Plugin System
Plugins extend the operation surface beyond base capabilities.

**Built-in: `tanstackQuery()` plugin:**
- `.queryOptions(input, opts?)` — for `useQuery()` / `useSuspenseQuery()`
- `.queryKey(input?)` — for cache invalidation
- `.mutationOptions(opts?)` — for `useMutation()`
- `.mutationKey()` — for `useMutationState()`
- `.infiniteQueryOptions(input, opts?)` — for `useInfiniteQuery()` (auto-detected from paginated endpoints)
- `.pathKey()` — on API namespace, invalidate all queries of an API

Pagination detection for `infiniteQueryOptions`:
- REST: detects `offset`/`limit`, `cursor`/`after`, `page`/`pageSize` params from OpenAPI spec
- GraphQL: detects `first`/`after`, `last`/`before` (Relay-style) from document variables

```typescript
plugins: [tanstackQuery()]
```

**Future plugins:**
```typescript
plugins: [apollo()]     // documentNode, watchQuery options
plugins: [zod()]        // .schema per operation (input/output validation)
plugins: [msw()]        // mock service worker handlers
```

### F-005: Runtime Client (`createMagia`)
tRPC v11-inspired runtime setup with Proxy-based namespace:

```typescript
// src/lib/magia.ts
import { createMagia } from 'magia-api'

export const magia = createMagia({
  apis: {
    petstore: {
      baseUrl: import.meta.env.VITE_PETSTORE_URL,
      fetchOptions: {
        headers: { 'X-Api-Key': import.meta.env.VITE_PETSTORE_KEY },
      },
    },
    github: {
      baseUrl: import.meta.env.VITE_GITHUB_GQL_URL,
      fetchOptions: {
        headers: () => ({ Authorization: `Bearer ${getToken()}` }),
      },
    },
  },
  // Global error handler (see F-013 for full error handling spec)
  onError: (error) => {
    Sentry.captureException(error)
    if (error.isAuthError()) authService.signOut()
  },
  // Optional: transform errors before they're thrown
  transformError: (error) => new ApplicationError(mapCode(error.code)),
})
```

**Usage — fetch (always available, same for REST and GraphQL):**
```typescript
// REST
const pet = await magia.petstore.getPetById.fetch({ petId: 1 })
// GraphQL
const user = await magia.github.getUser.fetch({ login: 'octocat' })
// With AbortSignal
const pet = await magia.petstore.getPetById.fetch({ petId: 1 }, { signal: controller.signal })
// With response headers
const { data, headers } = await magia.petstore.listPets.fetch({ limit: 10 }, { raw: true })
```

**Usage — subscriptions (same API for REST SSE and GraphQL subscriptions):**
```typescript
// REST SSE (auto-detected from text/event-stream in OpenAPI spec)
const stream = magia.ai.streamChat.subscribe({ message: 'hello' })
for await (const event of stream) {
  console.log(event) // fully typed
}

// GraphQL subscription (via graphql-request SSE transport)
const stream = magia.github.onIssueCreated.subscribe({ repo: 'my-repo' })
for await (const event of stream) {
  console.log(event) // fully typed
}

// Both support abort + reconnection
const stream = magia.ai.streamChat.subscribe(
  { message: 'hello' },
  { signal: controller.signal, reconnect: true }
)
```

**Usage — file uploads (auto-detected from multipart/form-data):**
```typescript
// Accepts File/Blob directly — magia constructs FormData internally
const result = await magia.petstore.uploadPetImage.fetch({
  petId: 1,
  file: fileInput.files[0],  // File object
})
```

**Usage — with `tanstackQuery()` plugin:**
```typescript
import { useQuery, useMutation, useInfiniteQuery, useQueryClient } from '@tanstack/react-query'

// Queries
const { data } = useQuery(magia.petstore.getPetById.queryOptions({ petId: 1 }))
const { data } = useQuery(magia.github.getUser.queryOptions({ login: 'octocat' }))

// Mutations
const { mutate } = useMutation(magia.petstore.createPet.mutationOptions())

// Infinite queries (auto-detected paginated endpoints)
const { data, fetchNextPage } = useInfiniteQuery(
  magia.petstore.listPets.infiniteQueryOptions({ limit: 10 })
)

// Cache control
const qc = useQueryClient()
qc.invalidateQueries({ queryKey: magia.petstore.getPetById.queryKey() })
qc.invalidateQueries({ queryKey: magia.petstore.pathKey() })

// Override inline
const { data } = useQuery({
  ...magia.petstore.getPetById.queryOptions({ petId: 1 }),
  staleTime: 30_000,
  enabled: !!petId,
})
```

**Parameter mapping** (flat object, auto-mapped from manifest):
```typescript
// User writes flat:
magia.petstore.getPetById.fetch({ petId: 1, status: 'available' })
// Manifest knows: petId → path, status → query
// Proxy splits automatically: GET /pet/1?status=available

// For edge cases, explicit mapping via options:
magia.petstore.createPet.fetch(
  { name: 'Rex', status: 'available' },
  { query: { dryRun: true }, headers: { 'X-Custom': 'value' } }
)
```

**Runtime Proxy pattern:**
- `createMagia()` returns a recursive Proxy that accumulates path segments
- Manifest bundled via `virtual:magia-manifest` Vite virtual module
- **REST dispatch**: uses `@hey-api/client-fetch` — constructs URL from path template + params
- **GraphQL dispatch**: uses `graphql-request` — sends query document + variables
- **SSE/subscriptions**: REST uses Hey API SSE, GraphQL uses graphql-request SSE transport
- **File uploads**: auto-constructs `FormData` from `File`/`Blob` inputs
- **Same dispatch interface** — the user never knows which protocol is underneath
- Runtime is untyped — `src/magia-api.d.ts` provides all type safety

**Query keys** (hierarchical, TQ plugin):
```typescript
// ["magia", "<api>", "<operation>", input?]
// e.g. ["magia", "petstore", "getPetById", { petId: 1 }]
// magia.petstore.pathKey() → ["magia", "petstore"]
```

### F-006: Type Generation (`.d.ts` augmentation)
Single gitignored file in `src/` — auto-generated, no tsconfig changes needed:

```typescript
// src/magia-api.d.ts (auto-generated, gitignored)
declare module 'magia-api' {
  interface MagiaClient {
    // REST API
    petstore: {
      getPetById: MagiaOperation<GetPetByIdParams, Pet, { 404: NotFoundError }>
        & MagiaTanStackQuery<GetPetByIdParams, Pet>
      createPet: MagiaMutation<CreatePetParams, Pet>
        & MagiaTanStackMutation<CreatePetParams, Pet>
      listPets: MagiaOperation<ListPetsParams, Pet[]>
        & MagiaTanStackQuery<ListPetsParams, Pet[]>
        & MagiaTanStackInfiniteQuery<ListPetsParams, Pet[]>
      uploadPetImage: MagiaMutation<UploadPetImageParams, void>
        & MagiaTanStackMutation<UploadPetImageParams, void>
      // SSE — auto-detected from text/event-stream
      watchPetUpdates: MagiaSSEOperation<WatchParams, PetUpdateEvent>
      pathKey(): readonly ['magia', 'petstore']
    }
    // GraphQL API — same surface
    github: {
      getUser: MagiaOperation<GetUserVariables, GetUserQuery>
        & MagiaTanStackQuery<GetUserVariables, GetUserQuery>
      createIssue: MagiaMutation<CreateIssueInput, CreateIssueMutation>
        & MagiaTanStackMutation<CreateIssueInput, CreateIssueMutation>
      // GraphQL subscription — same .subscribe() as REST SSE
      onIssueCreated: MagiaSSEOperation<OnIssueCreatedVars, IssueCreatedEvent>
      pathKey(): readonly ['magia', 'github']
    }
    // No plugins — fetch only
    payments: {
      getInvoice: MagiaOperation<GetInvoiceParams, Invoice>
    }
  }
}

// Base types (shipped with magia-api package)
interface MagiaOperation<TInput, TOutput, TErrors = {}> {
  fetch(input: TInput, opts?: MagiaFetchOptions): Promise<TOutput>
  fetch(input: TInput, opts: MagiaFetchOptions & { raw: true }): Promise<MagiaRawResponse<TOutput>>
  isError<TCode extends keyof TErrors>(
    error: unknown, code: TCode
  ): error is MagiaError & { status: TCode; data: TErrors[TCode] }
}
interface MagiaMutation<TInput, TOutput, TErrors = {}> {
  fetch(input: TInput, opts?: MagiaFetchOptions): Promise<TOutput>
  isError<TCode extends keyof TErrors>(
    error: unknown, code: TCode
  ): error is MagiaError & { status: TCode; data: TErrors[TCode] }
}
interface MagiaFetchOptions {
  signal?: AbortSignal
  raw?: boolean
  query?: Record<string, unknown>   // explicit query params (edge cases)
  headers?: Record<string, string>  // explicit headers (edge cases)
}
interface MagiaRawResponse<T> {
  data: T
  headers: Headers
  status: number
}

// SSE / Subscription types (core — auto-detected)
interface MagiaSSEOperation<TInput, TEvent, TErrors = {}> {
  subscribe(input: TInput, opts?: MagiaSSEOptions): AsyncIterable<TEvent>
  fetch(input: TInput, opts?: MagiaFetchOptions): Promise<TEvent>
}
interface MagiaSSEOptions {
  signal?: AbortSignal
  reconnect?: boolean
  lastEventId?: string
}

// tanstackQuery() plugin types
interface MagiaTanStackQuery<TInput, TOutput> {
  queryOptions(input: TInput, opts?: Partial<UseQueryOptions<TOutput>>): UseQueryOptions<TOutput>
  queryKey(input?: TInput): readonly ['magia', string, string, TInput?]
}
interface MagiaTanStackMutation<TInput, TOutput> {
  mutationOptions(opts?: Partial<UseMutationOptions<TOutput, Error, TInput>>): UseMutationOptions<TOutput, Error, TInput>
  mutationKey(): readonly ['magia', string, string]
}
interface MagiaTanStackInfiniteQuery<TInput, TOutput> {
  infiniteQueryOptions(input: TInput, opts?: Partial<UseInfiniteQueryOptions<TOutput>>): UseInfiniteQueryOptions<TOutput>
}

// Type inference utilities
type InferInput<TApi extends keyof MagiaClient, TOp extends keyof MagiaClient[TApi]> = ...
type InferOutput<TApi extends keyof MagiaClient, TOp extends keyof MagiaClient[TApi]> = ...
```

- One file: `src/magia-api.d.ts` (gitignored, auto-generated)
- No tsconfig changes — `src/` is already in `include`
- Atomic writes + mtime cache to prevent watcher loops
- Plugin types intersected only when plugin is configured

## Phase 2: Vite Plugin — Seamless DX

### F-008: Vite Plugin
- Reads `magia-api.config.ts`
- Triggers codegen on server start
- Resolves `virtual:magia-manifest` for runtime bundling
- Watches schema sources for changes (F-002 strategy)
- Re-generates on schema change
- Shows clear errors in Vite overlay on generation failure

```typescript
// vite.config.ts
import { magiaApi } from 'magia-api/vite'

export default defineConfig({
  plugins: [magiaApi()],
})
```

### F-009: Watch & Incremental
- Only re-generate APIs whose schema changed (hash comparison)
- Cache previous generation to avoid unnecessary rebuilds
- Atomic file writes to prevent partial reads during HMR

### F-014: Testing
Full testing strategy for developing magia-api + utilities shipped to users. See [full spec](../features/testing/spec.md).

**Developing magia-api:**
- Unit tests (native, fast): schema resolver, manifest gen, proxy, param mapper, query keys, errors
- Integration tests (native): full pipeline config → codegen → manifest → .d.ts, snapshot output
- E2E tests (Docker, isolated): 10 states — each a real Vite project in its own container
- Type tests: `expectTypeOf` for .d.ts augmentation correctness
- CI: unit native → E2E Docker (parallel states)

**User-facing testing utilities** shipped with `magia-api/test`:

**Tier 1 — Full mock client:**
```typescript
import { createTestMagia } from 'magia-api/test'

const magia = createTestMagia({
  petstore: {

```

**Tier 2 — Per-operation mock/spy:**
```typescript
import { mockOperation } from 'magia-api/test'

mockOperation(magia.petstore.getPetById, {

// Or with function:
mockOperation(magia.petstore.getPetById, (input) => {
  if (input.petId === 999) throw new MagiaError({ status: 404 })

```

**Tier 3 — Faker mocks (auto-generated from schema):**
```typescript
import { createFakerMagia } from 'magia-api/test'

// Auto-generates realistic fake data from OpenAPI/GraphQL schema types
const magia = createFakerMagia()

const pet = await magia.petstore.getPetById.fetch({ petId: 1 })
// → { id: 42, name: 'Fluffy', status: 'available', photoUrls: ['https://...'] }
// All fields populated with faker data matching schema types/formats

// Seed for deterministic tests
const magia = createFakerMagia({ seed: 123 })

// Override specific operations while keeping faker for the rest
const magia = createFakerMagia({
  overrides: {
    petstore: {
      getPetById: { data: { id: 1, name: 'Rex' } },  // explicit
    },
  },
})
```

Uses `@faker-js/faker` to generate data based on schema types:
- `string` → `faker.lorem.word()`
- `string (email)` → `faker.internet.email()`
- `string (date-time)` → `faker.date.recent().toISOString()`
- `integer` → `faker.number.int()`
- `boolean` → `faker.datatype.boolean()`
- `array` → generates 1-3 items
- `$ref` → recursively generates nested objects
- GraphQL types mapped the same way from schema scalars

### Key v1 changes from original design

- **Manifest**: Real file `src/magia.gen.ts` (TanStack Router-style), not virtual module. User imports it explicitly.
- **Plugins**: Compile-time only in `defineConfig()`, not passed to `createMagia()`. Manifest carries plugin metadata.
- **Runtime client**: Native `fetch` instead of `@hey-api/client-fetch` (deprecated).
- **createMagia(config, manifest)**: Takes manifest explicitly. Works in Vite, Node, any bundler.
